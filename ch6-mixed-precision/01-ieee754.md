# 01. IEEE 754 — FP32 / FP16 / BF16 / TF32

## 🎯 핵심 질문

- IEEE 754 표준에서 부동소수점 수는 (sign, exponent, mantissa) 3개 필드로 표현되는데, 각 타입이 비트를 어떻게 분배하는가?
- FP32 (1+8+23) 의 정규화 범위와 FP16 (1+5+10) 의 범위가 왜 $10^{23}$ 배 차이가 나는가?
- BF16 은 같은 exponent bit 수로 FP32 의 range 를 유지하면서 어떻게 mantissa 를 포기하는가?
- TF32 (A100+) 가 왜 "transparent" 라 불리고 FP32 input 에서 내부적으로 무엇이 일어나는가?
- Subnormal (denormalized) 수, NaN, Infinity 의 정의와 ML 에서의 의미는?

---

## 🔍 왜 이 내부 구조가 PyTorch 사용자에게 중요한가

Mixed precision training 의 출발점은 **수에 대한 엄밀한 이해** 입니다. 많은 사용자가 `torch.autocast()` 또는 `torch.cuda.amp` 로 AMP 를 켜면 "빨라진다" 고만 압니다. 그러나 다음을 정확히 모르고 사용합니다:

1. **FP16 의 range 제약** — ML 의 typical gradient ($10^{-7}$ ~ $10^{-3}$) 중 상당 부분이 FP16 의 normal range ($6.1 \times 10^{-5}$ 이상) 미만 → **silent underflow**
2. **Loss scaling 이 필수인 이유** — 수치 분석적으로 gradient 를 scale 하면 representable range 로 끌어올릴 수 있다는 증명
3. **BF16 vs FP16** — 왜 modern LLM (Llama, Mistral, ...) 이 FP16 대신 BF16 을 기본으로 사용하는가
4. **TF32 의 trade-off** — A100+ 의 자동 가속이 왜 정확한지, 언제 문제가 될 수 있는지

이 문서는 IEEE 754 의 정확한 정의부터 각 타입의 bit layout, Python 에서의 확인 방법까지 다룹니다.

## 📖 직관적 이해

### 고정점 vs 부동소수점

**고정점 (Fixed-point)**: $x = \text{정수부} + 0.\text{소수부}$ — 범위는 $[0, 2^n)$ 로 고정, 정밀도는 균일.

**부동소수점 (Floating-point)**: $x = \text{sign} \times 1.\text{mantissa} \times 2^{\text{exponent}}$ — **범위는 큼** (exponent 덕분), 하지만 **정밀도는 상대적** (mantissa bit 수 고정, 절대값이 커질수록 단위가 커짐).

### 3개 필드의 역할

```
┌─────┬────────────┬──────────────────┐
│ S   │ E          │ M                │
│ 1b  │ e bits     │ m bits           │
└─────┴────────────┴──────────────────┘
  ↓        ↓                 ↓
sign   exponent         mantissa
       (bias)          (fraction)
       
정규화 수: ±1.M × 2^(E-bias)
```

- **Sign (S)**: 0=양수, 1=음수
- **Exponent (E)**: bias 를 뺀 후 $2^E$ 를 결정 (bias = $2^{e-1}-1$)
- **Mantissa (M)**: 소수점 이하 부분, implicit leading 1

### FP32 vs FP16 의 범위 차이

| 타입 | 구조 | Bias | Min Normal | Max |
|------|------|------|------------|-----|
| FP32 | 1+8+23 | 127 | $2^{-126} \approx 1.2 \times 10^{-38}$ | $2^{128} \approx 3.4 \times 10^{38}$ |
| FP16 | 1+5+10 | 15 | $2^{-14} \approx 6.1 \times 10^{-5}$ | $2^{16} \approx 6.5 \times 10^{4}$ |

**차이 원인**: exponent bit 수 차이 (8 vs 5) → $2^{8-5} = 2^3 = 8$ 배, 수치로는 지수의 range 가 기하급수

## ✏️ 엄밀한 정의

### 정의 1.1 — IEEE 754 부동소수점 수

정규화된 부동소수점 수는:
$$x = (-1)^S \times 1.M \times 2^{E - \text{bias}}$$

여기서:
- $S \in \{0, 1\}$ (sign bit)
- $E$ 는 exponent field, $0 < E < 2^e - 1$ (정규화 조건)
- $M$ 는 mantissa의 fractional part, $0 \leq M < 1$ (implicit leading 1 가정)
- bias $= 2^{e-1} - 1$ (exponent 중심 이동)

### 정의 1.2 — 각 타입의 파라미터

| 타입 | Sign | Exponent | Mantissa | Bias | Min Normal | Max Normal | Precision |
|------|------|----------|----------|------|------------|------------|-----------|
| **FP32** | 1 | 8 | 23 | 127 | $1.2 \times 10^{-38}$ | $3.4 \times 10^{38}$ | ~6–7 decimal |
| **FP16** | 1 | 5 | 10 | 15 | $6.1 \times 10^{-5}$ | $6.5 \times 10^{4}$ | ~3–4 decimal |
| **BF16** | 1 | 8 | 7 | 127 | $1.2 \times 10^{-38}$ | $3.4 \times 10^{38}$ | ~2–3 decimal |
| **TF32** | 1 | 8 | 10 | 127 | $1.2 \times 10^{-38}$ | $3.4 \times 10^{38}$ | ~4–5 decimal |

### 정의 1.3 — Subnormal (Denormalized) 수

정규화 범위 미만을 표현하기 위해 implicit leading 1 을 포기:
$$x_{\text{subnormal}} = (-1)^S \times 0.M \times 2^{\text{bias} - (e-1)}$$

예: FP16 에서 가장 작은 subnormal $= 2^{-24} \approx 5.96 \times 10^{-8}$ (normal min $2^{-14}$ 보다 작음)

### 정의 1.4 — Special Values

| 경우 | Exponent | Mantissa | 값 |
|------|----------|----------|-----|
| Zero | 모두 0 | 모두 0 | ±0 |
| Subnormal | 모두 0 | 0이 아님 | $\pm 0.M \times 2^{\text{min exp}}$ |
| Normal | 0과 $2^e-1$ 사이 | 임의 | ±1.M × 2^(E-bias) |
| Infinity | 모두 1 | 모두 0 | ±∞ |
| NaN | 모두 1 | 0이 아님 | Not a Number |

## 🔬 정리와 증명

### 정리 1.1 — 정규화 수의 표현 범위

FP32 의 정규화 범위는:
$$\boxed{\left[\, -2^{128} + \epsilon,\; -2^{-126} \right] \cup \left[\, 2^{-126},\; 2^{128} - \epsilon \,\right]}$$

여기서 $\epsilon$ 는 mantissa 분해능 (ulp, unit in the last place).

**증명**:
- Min positive normal: $E = 1 \Rightarrow 2^{1-127} = 2^{-126}$ (implicit 1.0)
- Max positive normal: $E = 254 \Rightarrow 2^{254-127} = 2^{127}$, mantissa $\approx 1.999... \Rightarrow$ max $\approx 2^{128} - 2^{104}$ $\square$

### 정리 1.2 — Machine Epsilon 과 상대오차

FP 수 $x$ 근처에서 연속된 두 representable 수의 차이 (ulp at $x$):
$$\text{ulp}(x) = 2^{\text{exponent}(x) - \text{mantissa\_bits}}$$

따라서 상대오차는:
$$\frac{\text{ulp}(x)}{x} = 2^{- \text{mantissa\_bits}} =: \epsilon_{\text{mach}}$$

| 타입 | Mantissa bits | Machine epsilon |
|------|---------------|-----------------|
| FP32 | 23 | $\approx 1.2 \times 10^{-7}$ |
| FP16 | 10 | $\approx 9.8 \times 10^{-4}$ |
| BF16 | 7 | $\approx 7.8 \times 10^{-3}$ |

**의미**: FP16 은 FP32 보다 **1000배 덜 정확함**. ML gradient ($10^{-3}$ 규모) 의 정확성이 심각하게 손상될 수 있음.

### 따름 정리 1.3 — FP16 Underflow 의 임계값

ML 에서 typical gradient: $\nabla L \sim 10^{-3}$ ~ $10^{-7}$.

FP16 normal range: $[\, 6.1 \times 10^{-5}, \, 6.5 \times 10^{4} \,]$.

따라서 **$\nabla L < 6.1 \times 10^{-5}$ 인 경우 0 으로 underflow** → 학습 불안정.

### 실험 2 — Underflow 재현

```python
import torch

# FP16 에서 small gradient 생성
grad = 1e-6

t_fp32 = torch.tensor(grad, dtype=torch.float32)
t_fp16 = torch.tensor(grad, dtype=torch.float16)

print(f"FP32 값:  {t_fp32.item():.2e}")
print(f"FP16 값:  {t_fp16.item():.2e}")
print(f"일치:     {(t_fp16.item() == 0.0)}")

# 1e-5 (FP16 range 내)

t_fp32_2 = torch.tensor(grad2, dtype=torch.float32)
t_fp16_2 = torch.tensor(grad2, dtype=torch.float16)

print(f"\nFP32 값:  {t_fp32_2.item():.2e}")
print(f"FP16 값:  {t_fp16_2.item():.2e}")
print(f"일치:     {(t_fp16_2.item() != 0.0)}")
```

### 실험 3 — Bit layout 직접 검사

```python
import torch
import struct

def fp32_to_bits(x):
    return format(struct.unpack('>I', struct.pack('>f', x))[0], '032b')

x = 1.5
bits = fp32_to_bits(x)
s = bits[0]
e = bits[1:9]
m = bits[9:]

print(f"1.5 = {x}")
print(f"  Sign:      {s}")
print(f"  Exponent:  {e} (0x{int(e,2):02x} = {int(e,2)} → 2^0)")
print(f"  Mantissa:  {m}")
print(f"  Result:    1.{m}_b × 2^0 = 1.5")
```

## 🔗 실전 활용

### 1. autocast 에서 자동 dtype 선택

`torch.autocast('cuda')` 를 활성화하면:
- **FP32 → FP16**: matmul, conv, linear
- **FP32 → BF16**: reduction ops
- **FP32 유지**: loss, normalization weights

### 2. 수동 scaling 과의 상호작용

```python
loss = criterion(output, target)
loss_scaled = loss * scale
loss_scaled.backward()
```

### 3. CUDA kernel 에서의 안전한 사용

```cuda
__global__ void kernel(const half *input) {
    float acc = 0.0f;  // FP32 accumulator
    for (int i = 0; i < N; ++i) {
        acc += __half2float(input[i]);
    }
}
```

## 🤔 생각해볼 문제

**문제 1** (기초): FP32 의 bias 가 127 인 이유를 설명하고, 일반적인 bias 공식을 유도하라.

<details>
<summary>해설</summary>

Bias 공식: $\text{bias} = 2^{e-1} - 1$.

**FP32**: $e = 8$ → bias $= 2^7 - 1 = 127$ ✓

**이유**: exponent field 가 unsigned integer 로 저장되지만, signed exponent 로 해석하기 위해 bias 를 뺀다. $2^{e-1} - 1$ 선택으로 음수와 양수 범위가 거의 대칭.

**예**: FP32 에서 지수는 $-126$ (E=1) ~ $+127$ (E=254). $\square$

</details>

**문제 2** (심화): FP16 에서 정규화 수와 subnormal 수 사이에 표현 불가능한 "gap" 이 있는가? 왜 그런가?

<details>
<summary>해설</summary>

**정규화 수**: $E \geq 1$ → $2^{E-15} \geq 2^{-14}$.
**Subnormal 수**: $E = 0$ → $0.M \times 2^{-14}$, $0 < M < 1$.

**Gap 없음** — subnormal 이 정규화와 0 사이를 seamlessly 채운다.

**이유**: subnormal 의 exponent 를 명시적으로 $2^{-(e-1)-1} = 2^{-15}$ 로 정의 (FP16: $2^{-15}$), 가장 큰 subnormal $(1 - 2^{-10}) \times 2^{-14} \approx 0.999 \times 6.1e-5$ 는 가장 작은 정규화 $1.0 \times 2^{-14} = 6.1e-5$ 에 거의 가깝다. $\square$

</details>

**문제 3** (논문 비평): Kalamkar et al. (2019) 의 "A Case for Mixed Precision Training" 에서 왜 BF16 이 FP16 보다 robust 하다고 주장했는가? IEEE 754 관점에서 설명하라.

<details>
<summary>해설</summary>

**이유**: Exponent bit 수.

- **FP16**: 5 bits → range $[\, 6.1 \times 10^{-5}, \, 6.5 \times 10^4 \,]$ (narrow)
- **BF16**: 8 bits → range $[\, 1.2 \times 10^{-38}, \, 3.4 \times 10^{38} \,]$ (FP32 동일)

ML gradient $10^{-5}$ ~ $10^{-7}$ 은 FP16 range 에 위험하게 가깝다. BF16 은 scaling 없이도 안전. $\square$

Kalamkar et al. 이 Google TPU 에서 BF16 의 이점을 처음 보였고, 현재 Llama, Mistral 등 modern LLM 이 BF16 을 기본으로 사용.

</details>
